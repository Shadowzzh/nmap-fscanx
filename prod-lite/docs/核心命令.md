# 核心命令

这里只写原生 `fscanx` 的两阶段用法。

约定：

- 第一阶段在 `phase1/` 目录执行
- 第二阶段在 `phase2/` 目录执行
- 第一阶段产出的 `alive_ips.txt` 交给第二阶段继续扫描
- `-json` 只改变结果内容格式
- 默认结果文件仍然是当前目录下的 `result.txt`

## 1. 第一阶段

作用：

- 用重点端口快速筛出命中主机
- 产出 `alive_ips.txt`

执行：

```bash
mkdir -p /tmp/fscanx-manual/phase1
cd /tmp/fscanx-manual/phase1

fscanx \
  -h '192.168.1.0/24,192.168.20.0/24,192.168.4.0/24' \
  -p 22,80,443,445,3389 \
  -np \
  -t 4000 \
  -time 1 \
  -nocolor \
  -json
```

执行后原始结果在：

```bash
result.txt
```

提取第一阶段命中的 IP：

```bash
awk '
  NF {
    line = $0
    sub(/,[[:space:]]*$/, "", line)
    lines[++count] = line
  }
  END {
    print "["
    for (i = 1; i <= count; i++) {
      suffix = ""
      if (i < count) {
        suffix = ","
      }
      print lines[i] suffix
    }
    print "]"
  }
' result.txt > normalized.json

jq -r '
  .[]
  | select(.type == "Port")
  | try (.text | capture("open\\t(?<ip>[0-9.]+):(?<port>[0-9]+)").ip) catch empty
' normalized.json | awk 'NF && !seen[$0]++' > alive_ips.txt
```

第一阶段产物：

- `result.txt`
- `normalized.json`
- `alive_ips.txt`

## 2. 第二阶段

作用：

- 读取第一阶段产出的 `alive_ips.txt`
- 对这些主机继续扫描

执行：

```bash
mkdir -p /tmp/fscanx-manual/phase2
cd /tmp/fscanx-manual/phase2

fscanx \
  -hf ../phase1/alive_ips.txt \
  -p 1-65535 \
  -np \
  -t 4000 \
  -time 1 \
  -nocolor \
  -json
```

第二阶段输入来自：

```bash
../phase1/alive_ips.txt
```

执行后原始结果在：

```bash
result.txt
```
